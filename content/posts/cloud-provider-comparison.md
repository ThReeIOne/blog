云厂商选型是企业基础设施决策中最重要的一步，选错了迁移成本极高。本文不写"入门使用教程"，而是从产品能力、性能、定价、合规、生态等维度做横向对比，帮你搞清楚不同场景下该选谁。

## 一、市场格局：先看清楚玩家

### 全球三大公有云
- **AWS（Amazon Web Services）**：市场份额第一（约 31%），产品最全，生态最成熟，1000+ 服务
- **Microsoft Azure**：市场份额第二（约 25%），企业/政府客户强，与 Microsoft 生态深度绑定
- **Google Cloud（GCP）**：市场份额第三（约 11%），大数据/AI/ML 能力突出，Kubernetes 发源地

### 中国主要云厂商
- **阿里云**：国内市场份额第一（约 36%），产品线最全，出海企业首选
- **腾讯云**：国内第二（约 17%），游戏/视频/社交场景强，微信生态加持
- **华为云**：国内第三（约 12%），政企/运营商市场强，自研芯片（鲲鹏/昇腾）优势
- **京东云、百度智能云**：细分场景有竞争力，整体规模偏小

本文重点对比：**AWS、Google Cloud、阿里云、腾讯云、华为云**。

## 二、核心服务能力对比

### 2.1 计算服务

| 服务类别 | AWS | Google Cloud | 阿里云 | 腾讯云 | 华为云 |
|---------|-----|-------------|-------|-------|-------|
| 虚拟机 | EC2 | Compute Engine | ECS | CVM | ECS |
| 容器服务（K8s） | EKS | GKE（最原生） | ACK | TKE | CCE |
| Serverless 函数 | Lambda | Cloud Functions | 函数计算 FC | SCF | FunctionGraph |
| 裸金属 | ✅ | ✅ | ✅ | ✅ | ✅（鲲鹏） |
| GPU 实例 | ✅ 最全 | ✅ TPU 独家 | ✅ | ✅ | ✅ 昇腾 NPU |

**点评：**
- **GKE** 是 Kubernetes 发源地，K8s 体验业界最好，升级、自动修复、Autopilot 模式领先
- **AWS EC2** 实例类型最多（700+），覆盖场景最广
- **华为云** 自研鲲鹏（ARM）和昇腾（AI）芯片，信创/国产化场景有政策优势
- **阿里云 ECS** 在国内延迟和带宽优化做得最好

### 2.2 对象存储

| 维度 | AWS S3 | Google Cloud Storage | 阿里云 OSS | 腾讯云 COS | 华为云 OBS |
|------|--------|---------------------|-----------|-----------|-----------|
| 可用性 SLA | 99.99% | 99.99% | 99.9975% | 99.99% | 99.9% |
| 数据持久性 | 11 个 9 | 11 个 9 | 11 个 9 | 11 个 9 | 11 个 9 |
| 冷热分层 | ✅（S3 Glacier） | ✅（Archive） | ✅（归档存储） | ✅（归档存储） | ✅ |
| 国内访问速度 | ❌（需走境外） | ❌ | ✅ 最优 | ✅ | ✅ |
| 全球 CDN 集成 | CloudFront | Cloud CDN | CDN + DCDN | CDN | CDN |

**点评：**
- 国内业务数据存储，**阿里云 OSS** 性价比和访问速度最优
- 全球多区域分发，**AWS S3 + CloudFront** 是行业标准
- **腾讯云 COS** 在国内视频、图片场景集成很好（腾讯云直播/点播生态）

### 2.3 数据库服务

| 数据库类型 | AWS | Google Cloud | 阿里云 | 腾讯云 | 华为云 |
|-----------|-----|-------------|-------|-------|-------|
| MySQL 托管 | RDS MySQL / Aurora | Cloud SQL | RDS MySQL / PolarDB | TDSQL-C | RDS MySQL / GaussDB |
| PostgreSQL | RDS PostgreSQL / Aurora | Cloud SQL / AlloyDB | RDS PG / PolarDB-PG | TDSQL-C | GaussDB |
| NoSQL（文档） | DynamoDB / DocumentDB | Firestore / Bigtable | MongoDB 兼容版 | MongoDB 兼容版 | DCS（Redis） |
| Redis | ElastiCache | Memorystore | Tair（Redis 增强版） | Redis | DCS |
| 数据仓库 | Redshift | BigQuery（最强） | AnalyticDB / MaxCompute | TDSQL-A / CDW | DWS（GaussDB DWS） |
| HTAP | Aurora | AlloyDB | PolarDB | TDSQL | GaussDB |

**点评：**
- **BigQuery** 是 Google Cloud 最大的杀手锏之一：Serverless 数据仓库，按查询付费，PB 级查询秒级出结果，数据分析场景无出其右
- **阿里云 PolarDB** 是国内 MySQL 兼容云原生数据库的标杆，自研存储引擎，读写分离极其成熟
- **AWS Aurora** 全球最广泛使用的云原生关系数据库，MySQL/PostgreSQL 兼容
- **华为 GaussDB** 在金融/运营商领域有合规优势（可离线部署私有化）

### 2.4 AI / 机器学习平台

| 平台 | AWS | Google Cloud | 阿里云 | 腾讯云 | 华为云 |
|-----|-----|-------------|-------|-------|-------|
| ML 平台 | SageMaker | Vertex AI | PAI | TI-ONE | ModelArts |
| 大模型服务 | Bedrock（Claude/Llama等） | Gemini API | 通义千问 API | 混元大模型 | 盘古大模型 |
| 视觉 AI | Rekognition | Vision AI | 视觉智能开放平台 | 人脸识别/OCR | 图像识别 |
| 语音 AI | Transcribe / Polly | Speech-to-Text | 语音识别 ASR | 语音识别 ASR | 语音交互 |
| GPU/AI 芯片 | A100/H100/Trainium | TPU v4/v5 | A100/A10 | A100/T4 | 昇腾 910B |

**点评：**
- **Google Cloud** 的 AI/ML 能力整体最强：BigQuery ML、Vertex AI、TPU 独家优势、TensorFlow 原生支持
- **AWS Bedrock** 多模型接入（Claude、Llama、Mistral）是企业 LLM 应用首选，不绑定单一模型
- **华为 ModelArts + 昇腾** 在国产 AI 芯片路线上是唯一完整方案，信创/国产化场景刚需
- **阿里云 PAI** 在国内 AI 训练推理场景成熟度高，配套通义千问生态

## 五、各厂商的核心优势与短板

### AWS
**核心优势：**
- 产品线最全（1000+ 服务），几乎所有场景都有原生方案
- 生态最成熟：第三方工具、SaaS 集成、合作伙伴网络无敌
- 全球覆盖最广：33 个地理区域，105 个可用区
- 文档、社区、人才储备最丰富

**短板：**
- 定价复杂，成本容易超预期（出口流量费用尤其贵）
- 国内访问需通过中国区（与全球账号隔离），服务阉割
- 控制台略显臃肿

---

### Google Cloud
**核心优势：**
- 数据分析/AI/ML 能力最强（BigQuery、Vertex AI、TPU）
- Kubernetes 发源地，GKE 体验最好
- 网络基础设施全球最优（自建海底光缆）
- 定价对数据密集型应用友好（BigQuery 按查询付费）

**短板：**
- 国内没有节点，中国大陆访问慢
- 企业服务/销售支持历史上口碑不好（有砍产品前科：Google+、Stadia...）
- 市场份额第三，部分中小企业不愿"押注"

---

### 阿里云
**核心优势：**
- 国内覆盖最广，延迟最低
- 产品线最全（国内云厂商中），和 AWS 对标产品基本都有
- 出海东南亚节点优势，华人出海场景首选
- 等保合规、政企资质齐全

**短板：**
- 全球覆盖不如 AWS/GCP，欧美节点较少
- 部分产品稳定性有历史性事故（2022年香港机房故障等）
- 控制台产品太多，体验有时割裂

---

### 腾讯云
**核心优势：**
- 游戏、视频直播、社交场景深度优化（腾讯自家业务验证）
- 微信/小程序生态集成（微信支付、小程序托管、TRTC 音视频）
- 国内内容分发/CDN 网络密度高

**短板：**
- 全球覆盖弱于阿里云
- 产品线相对不如阿里云全
- 企业客户支持体验参差不齐

---

### 华为云
**核心优势：**
- **信创/国产化场景唯一完整方案**：鲲鹏（ARM）+ 昇腾（AI 芯片）全栈自研
- 政府、央企、金融、运营商客户首选（有政策背书）
- GaussDB 数据库在金融合规场景非常强
- 支持私有云/混合云部署（OpenStack 体系成熟）

**短板：**
- 公有云互联网场景生态不如阿里云/腾讯云
- 开发者友好度和文档质量有待提升
- 全球节点覆盖弱

## 六、选型决策框架

### 场景一：纯国内业务

```
业务类型？
├─ 游戏/视频/直播/小程序 → 腾讯云（生态绑定优势明显）
├─ 电商/零售/互联网 → 阿里云（最成熟，稳定性好）
├─ 政府/央企/金融/医疗 → 华为云或阿里云（合规资质最强）
└─ 普通 SaaS/API 服务 → 阿里云或腾讯云（按价格/团队熟悉度选）
```

### 场景二：出海业务

```
目标市场？
├─ 东南亚（新加坡/印尼/泰国） → 阿里云（节点多，华人生态）
├─ 欧美为主 → AWS（最成熟）或 GCP（数据分析强）
├─ 全球多地区 → AWS（覆盖最广）
└─ 日韩 → AWS 或阿里云均可
```

### 场景三：AI/大数据业务

```
业务重心？
├─ 数据仓库/BI 分析 → Google Cloud（BigQuery 无敌）
├─ 模型训练（大规模） → AWS（H100 资源多）或华为云（昇腾，信创路线）
├─ LLM 应用接入 → AWS Bedrock（多模型）
├─ 国内 AI 合规场景 → 阿里云（通义千问 API）
└─ 信创 AI → 华为云（昇腾 910B）
```

### 场景四：混合云/私有云

```
诉求？
├─ 与现有 Microsoft 生态整合 → Azure（未在本文重点讨论）
├─ 国产化/离线部署 → 华为云（OpenStack 体系，私有云成熟）
├─ 公私混合，统一管控 → 阿里云（ACK Anywhere / 本地集群）
└─ 全球统一 → AWS Outposts 或 GCP Distributed Cloud
```

## 七、总结

没有"最好的云"，只有"最适合你的云"。几条实用原则：

1. **国内业务首选阿里云或腾讯云**，别被 AWS/GCP 的品牌溢价迷惑，国内访问延迟差距是真实的
2. **出海业务别只用国内云**，AWS 在全球的基础设施和生态优势不是堆机器能追上的
3. **AI/大数据场景认真考虑 GCP**，BigQuery 和 Vertex AI 的能力差距是量级级别的
4. **政企/信创场景看华为云**，芯片自研 + 政策背书是其他厂商学不来的
5. **不要把鸡蛋放在一个篮子里**：核心数据库一个云，CDN 另一个，对象存储再一个——多云策略降低单点风险
6. **定价要算 TCO（总拥有成本）**，光看实例价格没意义，出口流量、数据迁移、技术支持费用都要算进去

最后：云厂商的服务能力在快速变化，今天的对比半年后可能就过时了。选型前最重要的一步是**把你的核心业务需求写清楚**，然后让各家云厂商 SA 来出方案，实际测试跑一遍，比看任何对比文章都靠谱。
